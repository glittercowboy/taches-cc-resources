Purpose: The Shiny app should be fast. All groupBy + summarise operations should run once at "data prep" time, not reactively. This plan produces a `dashboard_data` list object that global.R will load once at startup.
Output: `data/processed/09_dashboard_data.rds` — a named list with pre-computed summary tables and the full dataset for reactive filtering.
</objective>

    Create `R/prepare_dashboard_data.R` with `prepare_dashboard_data(df_full)` that builds and returns a named list:

    **`$records`** — the full tidy dataset (passed through, used for reactive filtering):
    Select only dashboard-needed columns to keep memory lean:
    `id, text, subreddit, keyword, type, created_utc, year, year_month, score, thread_depth, aspect_id, aspect_label, is_novel, aspect_sentiment, evidence, confidence, text_sentiment, emotion_label, sentiment_positive, sentiment_negative, sentiment_neutral`

    - `$subreddits` — sorted unique subreddit values
    - `$keywords` — sorted unique keyword values
    - `$aspects` — sorted unique non-NA aspect_label values (canonical + novel)
    - `$sentiments` — c("positive", "negative", "neutral", "mixed")
    - `$emotions` — sorted unique emotion_label values
    - `$years` — sorted unique year values

    **`$summary_by_aspect`** — grouped summary tibble: `aspect_label`, `n_records`, `pct_positive`, `pct_negative`, `pct_neutral`, `pct_mixed`

    **`$summary_by_subreddit`** — grouped summary: `subreddit`, `n_records`, `n_aspects`, most common `top_aspect`, most common `top_sentiment`

    **`$summary_by_month`** — time series: `year_month` (Date floor), `n_records`, `pct_positive`, `pct_negative`

    **`$aspect_sentiment_matrix`** — aspect × sentiment count matrix (for heatmap): cross-tab of `aspect_label` × `aspect_sentiment`

    **`$top_emotion_by_aspect`** — for each aspect, the top 3 emotions by frequency

    **`$metadata`** — list: `n_total_records`, `n_unique_ids`, `date_range`, `n_subreddits`, `n_aspects`, `pipeline_run_date = Sys.time()`
