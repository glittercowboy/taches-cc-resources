  <files>

    data/checkpoints/.gitkeep,
    R/utils.R,
    scripts/.gitkeep,
    app/.gitkeep,
    config/.gitkeep,
    .gitignore

  <action>
    Create the directory tree:
    - `data/raw/` — user places their source CSV(s) here
    - `data/processed/` — pipeline outputs at each stage
    - `data/checkpoints/` — intermediate saves for resumable LLM batching
    - `R/` — reusable functions sourced by scripts
    - `scripts/` — numbered pipeline execution scripts (01_filter.R, 02_relevance.R etc.)
    - `app/` — Shiny dashboard
    - `config/` — configuration files (NOT committed if containing secrets)

    Create `.gitignore` with entries: `.Renviron`, `config/secrets.R`, `data/raw/*`, `data/processed/*`, `data/checkpoints/*`, `!data/raw/.gitkeep`, `!data/processed/.gitkeep`, `!data/checkpoints/.gitkeep`.

    Create `R/utils.R` with:
    - `load_config()` — sources `config/config.R` and sets global options
    - `log_msg(msg, level = "INFO")` — timestamps to console (format: `[YYYY-MM-DD HH:MM:SS] [LEVEL] msg`)
    - `save_checkpoint(data, name)` — saves RDS to `data/checkpoints/{name}_{timestamp}.rds`
    - `load_latest_checkpoint(name)` — loads most recent checkpoint matching name pattern
  </action>
  <verify>Running `source("R/utils.R")` in R console produces no errors; `dir.exists("data/raw")` returns TRUE</verify>
  <done>All directories exist; `R/utils.R` sources cleanly; `.gitignore` excludes secrets and data files</done>
</task>

      # Paths (use here::here() for portability)
