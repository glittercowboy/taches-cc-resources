<objective>
Use Claude (via ellmer) to classify each discussion excerpt as relevant or not relevant to AI use in healthcare, using the full thread context string built in Plan 01-03.

Purpose: Rule-based keyword matching is insufficient for relevance — a comment may be relevant because its parent post discusses AI diagnostics, even if the comment itself uses no keywords. LLM classification with thread context solves this. Checkpointing is required because 5k–50k LLM API calls must be resumable.
Output: `data/processed/03_relevance_classified.rds` — all records with `is_relevant` (logical) and `relevance_reason` (character) columns; only relevant records retained in final output.
</objective>

    **1. Relevance classification prompt** (`build_relevance_prompt(full_context)`):
    System prompt (set once):
    ```
    You are classifying Reddit discussion excerpts for relevance to AI use in healthcare.
    Relevant discussions address: AI/ML tools in clinical settings, medical diagnosis/imaging AI,
    clinical decision support systems, AI in drug discovery, healthcare data/EHR AI, robotic surgery AI,
    AI regulation in medicine, bias in medical AI, clinician attitudes toward AI tools, patient safety

    NOT relevant: general AI discussions without healthcare context, general healthcare discussions
    ```

    User message: the `full_context` string for this record.

    **2. `classify_single(record, chat_obj)`** — calls the LLM, parses JSON response, returns list(is_relevant, confidence, reason). On JSON parse failure, returns list(is_relevant=NA, confidence="error", reason="parse_error").

    **3. `classify_relevance_batch(df, cfg = CFG)`** — main batching function:
    - Check for existing checkpoint with `load_latest_checkpoint("relevance")`; if found, skip already-classified IDs
    - Process in batches of 100 records
    - Use a single `ellmer::chat_anthropic()` session with `model = cfg$llm_model`, `temperature = cfg$llm_temperature`
    - After each batch: `save_checkpoint(results_so_far, "relevance")`
    - Show progress with `progressr::with_progress()`
    - Rate limit: add `Sys.sleep(0.1)` between individual calls (avoid 429s on free tier); increase to 0.5 if 429 detected
    - Log estimated cost: Claude Sonnet input ~$3/M tokens. Estimate ~200 tokens per classification. Log running cost estimate every 500 records.
    - Return tibble with columns: `id`, `is_relevant`, `relevance_confidence`, `relevance_reason`

    Do NOT use `future`/parallel here — sequential is safer for LLM rate limits and easier to debug.

  <action>
    Create `scripts/03_classify_relevance.R`:
    ```r
    source("R/utils.R"); load_config()
    source("R/classify_relevance.R")

    df_threaded <- readRDS(file.path(CFG$path_processed, "02_with_thread_context.rds"))
    log_msg(paste("Starting relevance classification for", nrow(df_threaded), "records"))

    relevance_results <- classify_relevance_batch(df_threaded)

    # Join results back
    df_classified <- df_threaded |>
      dplyr::left_join(relevance_results, by = "id")

    # Summary
    n_relevant   <- sum(df_classified$is_relevant == TRUE, na.rm = TRUE)
    n_irrelevant <- sum(df_classified$is_relevant == FALSE, na.rm = TRUE)
    n_error      <- sum(is.na(df_classified$is_relevant))
    log_msg(paste("Relevance results — Relevant:", n_relevant,
                  "| Not relevant:", n_irrelevant, "| Errors:", n_error))

    # Save full results (including non-relevant, for audit)
    saveRDS(df_classified, file.path(CFG$path_processed, "03_relevance_classified_all.rds"))

    # Save relevant-only for downstream pipeline
    df_relevant <- dplyr::filter(df_classified, is_relevant == TRUE)
    saveRDS(df_relevant, file.path(CFG$path_processed, "03_relevance_classified.rds"))
    log_msg(paste("Saved", nrow(df_relevant), "relevant records to 03_relevance_classified.rds"))
    ```
