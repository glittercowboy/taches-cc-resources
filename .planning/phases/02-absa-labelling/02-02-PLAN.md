<task type="auto">
  <name>Task 1: Create R/run_absa.R — batch ABSA pipeline with checkpointing</name>
  <files>R/run_absa.R</files>
  <action>
    Create `R/run_absa.R` with `run_absa_batch(df, cfg = CFG)`:

    **Core loop:**
    1. Load checkpoint: `load_latest_checkpoint("absa")` → get already-processed IDs; skip them
    2. Create a single `ellmer::chat_anthropic()` instance with `system = build_absa_system_prompt()`, `model = cfg$llm_model`, `api_key = cfg$anthropic_api_key`
    3. Process records one at a time (NOT parallel — preserves conversation context and avoids rate limits):
       - Call `chat$chat(build_absa_user_message(record))` — note: for ABSA, a FRESH chat per record is needed (no multi-turn memory wanted). Create a new chat object each time: `ellmer::chat_anthropic(system = sys_prompt, model = model)`.
       - Attempt `jsonlite::fromJSON(response_text, simplifyVector = FALSE)`
       - On success: store parsed list in results
       - On JSON parse failure: store `list(aspects = list(), parse_error = TRUE, raw_response = response_text)`
       - On API error (caught with `tryCatch`): store `list(aspects = list(), api_error = TRUE)`, wait 5s, continue
    4. Every 100 records: `save_checkpoint(results_so_far, "absa")`
    5. Log progress every 100 records: "Processed X/N | Errors: Y | Est. cost: $Z"
    6. Cost estimation: ~300 tokens/record (input system prompt ~500 + user ~200 + output ~100). Claude Sonnet: ~$3/M input, ~$15/M output. Log running estimate.
    7. Rate limiting: `Sys.sleep(0.1)` between calls. On HTTP 429: exponential backoff starting at 5s.

    Return tibble: `id`, `absa_result` (list-column of parsed JSON), `absa_parse_error` (logical), `absa_api_error` (logical).

    Helper `validate_absa_result(result)`: checks that `result$aspects` is a list, each element has `aspect_id`, `sentiment`, `evidence`, `confidence`. Returns TRUE/FALSE.
  </action>
  <verify>`source("R/run_absa.R")` loads cleanly; `validate_absa_result(list(aspects = list()))` returns TRUE (empty is valid)</verify>
  <done>Functions load cleanly; validation helper works on both valid and malformed input</done>

    Create `scripts/05_run_absa.R`:
    ```r
    source("R/utils.R"); load_config()
    source("R/aspect_taxonomy.R")
    source("R/absa_prompt.R")
    source("R/run_absa.R")

    DRY_RUN <- FALSE  # Set TRUE to test on 30 records before full run

    df_human <- readRDS(file.path(CFG$path_processed, "04_human_filtered.rds"))
    if (DRY_RUN) df_human <- head(df_human, 30)

    log_msg(paste("Starting ABSA labelling for", nrow(df_human), "records"))

    # Cost estimate upfront
    est_cost <- nrow(df_human) * 800 / 1e6 * 3  # rough: 800 tokens/record at $3/M
    log_msg(paste("Estimated cost (rough): $", round(est_cost, 2)))

    absa_results <- run_absa_batch(df_human)

    # Join back
    df_absa <- df_human |> dplyr::left_join(absa_results, by = "id")

    # Summary
    n_ok    <- sum(!df_absa$absa_parse_error & !df_absa$absa_api_error, na.rm = TRUE)
    n_perr  <- sum(df_absa$absa_parse_error == TRUE, na.rm = TRUE)
    n_aerr  <- sum(df_absa$absa_api_error == TRUE, na.rm = TRUE)
    log_msg(paste("ABSA complete — OK:", n_ok, "| Parse errors:", n_perr, "| API errors:", n_aerr))

    saveRDS(df_absa, file.path(CFG$path_processed, "05_absa_raw.rds"))
    log_msg(paste("Saved raw ABSA output:", nrow(df_absa), "records"))
    ```
