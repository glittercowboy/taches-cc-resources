<objective>
Define the hybrid ABSA aspect taxonomy and design the LLM prompt that extracts aspect labels from discussion excerpts.

Purpose: The quality of the entire ABSA pipeline depends on prompt design. The taxonomy defines what aspects to look for; the prompt instructs the LLM to assign labels from the seed list OR propose novel aspects, returning structured JSON. This plan produces the taxonomy definition and a validated, tested prompt before batch execution begins.
Output: `R/aspect_taxonomy.R` — taxonomy definition; `R/absa_prompt.R` — prompt builder + single-record tester; validated JSON output format confirmed on sample records.
</objective>

<task type="auto">
  <name>Task 2: Create R/absa_prompt.R — ABSA prompt builder and single-record tester</name>
  <files>R/absa_prompt.R</files>
  <action>
    Create `R/absa_prompt.R` with:

    **1. `build_absa_system_prompt()`** — returns the system prompt string:
    ```
    You are an expert at Aspect-Based Sentiment Analysis (ABSA) for discussions about AI in healthcare.

    Your task: Given a Reddit discussion excerpt, identify which healthcare-AI aspects are discussed and the sentiment expressed toward each aspect.

    ASPECT TAXONOMY:
    {formatted taxonomy from format_taxonomy_for_prompt()}

    INSTRUCTIONS:
    1. Identify all aspects explicitly or implicitly discussed in the text.
    2. For each aspect, determine the sentiment: "positive", "negative", "neutral", or "mixed".
    3. If an aspect is clearly discussed but does not fit any taxonomy label, propose a novel label (short, title-case, ≤ 5 words) and mark it as novel.
    4. If no relevant aspect is discussed (e.g. the text is a brief acknowledgment or off-topic), return an empty aspects array.
    5. A single excerpt can have MULTIPLE aspects.

    RESPOND WITH VALID JSON ONLY — no markdown code fences, no explanation outside JSON:
    {
      "aspects": [
        {
          "aspect_id": "taxonomy_id_or_novel",
          "aspect_label": "Exact Label From Taxonomy or Novel Label",
          "is_novel": false,
          "sentiment": "positive" | "negative" | "neutral" | "mixed",
          "evidence": "brief quote or paraphrase from text supporting this label",
          "confidence": "high" | "medium" | "low"
        }
      ]
    }
    ```

    **2. `build_absa_user_message(record)`** — formats a single record for the user message:
    ```
    Subreddit: r/{record$subreddit}
    Keyword context: {record$keyword}

    {record$text}
    ```
    (Use `text` not `full_context` for ABSA — we want sentiment on the excerpt itself, not parent context. Thread context was needed for relevance only.)

    **3. `test_absa_single(record, cfg = CFG)`** — for manual testing:
    - Creates `ellmer::chat_anthropic()` with system prompt
    - Calls `chat$chat(build_absa_user_message(record))`
    - Attempts `jsonlite::fromJSON()` on response
    - Prints both raw response and parsed result
    - Returns parsed list or NULL on error

    **Few-shot note**: The system prompt alone is sufficient for Claude Sonnet. If quality is poor after dry run, few-shot examples can be added to `build_absa_system_prompt()`.
  </action>
  <verify>`source("R/absa_prompt.R"); source("R/aspect_taxonomy.R")` — both load cleanly; `build_absa_system_prompt()` returns a string containing "ASPECT TAXONOMY" and all 8 labels; `build_absa_user_message(test_record)` returns formatted string</verify>
