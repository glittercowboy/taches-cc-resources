  <files>R/integrate_results.R</files>
  <action>
    Create `R/integrate_results.R` with:

    **1. `load_transformer_scores(parquet_path)`**:
    - Read parquet with `arrow::read_parquet(parquet_path)`
    - Validate: `id` column present; at least `sentiment_label`, `emotion_label`, `sentiment_positive`, `sentiment_negative`, `sentiment_neutral` present
    - Log column names and row count
    - Ensure `id` is character type (to match R's id column)
    - Return tibble

    **2. `join_pipeline_results(df_absa_labelled, df_scores)`**:
    - Left join `df_absa_labelled` by `id` — keeps all ABSA rows, adds transformer scores
    - Since ABSA is one-row-per-aspect and transformer scores are one-row-per-record, the join will duplicate transformer scores across aspect rows for the same record. This is correct and expected.
    - Validate join: check for IDs in ABSA that are missing from scores (log count); check for unexpected NAs in `sentiment_label` after join
    - Log: "Join complete: X rows | Y records with transformer scores | Z records missing scores"

    **3. `validate_final_dataset(df)`**:
    Checks the following and logs PASS/FAIL for each:
    - [ ] `id` is non-null for all rows
    - [ ] `text` is non-null for all rows
    - [ ] `subreddit` has reasonable values (no NA > 5%)
    - [ ] `created_utc` is valid date/time
    - [ ] `sentiment_label` is in c("positive","negative","neutral") where present
    - [ ] `emotion_label` is non-null where transformer scores available
    - [ ] At least one row per unique `id` with a non-NA aspect

    **4. `summarise_pipeline(df)`**: Prints a summary:
    - Total unique records
    - Records with ≥1 aspect labelled
    - Aspect label distribution
    - Sentiment label distribution
    - Top 5 emotion labels
    - Subreddit distribution (top 10)
    - Date range

<task type="auto">
  <name>Task 2: Create scripts/07_integrate_results.R and produce final pipeline output</name>
  <files>scripts/07_integrate_results.R</files>
  <action>
    Create `scripts/07_integrate_results.R`:
    ```r
    source("R/utils.R"); load_config()
    source("R/integrate_results.R")

    # Load inputs
    df_absa <- readRDS(file.path(CFG$path_processed, "06_absa_labelled.rds"))
    df_scores <- load_transformer_scores(
      file.path(CFG$path_processed, "07_transformer_scores.parquet")
    )

    # Join
    df_final <- join_pipeline_results(df_absa, df_scores)

    # Validate
    validate_final_dataset(df_final)

    # Add convenience columns for dashboard
    df_final <- df_final |>
      dplyr::mutate(
        year       = lubridate::year(created_utc),
        month      = lubridate::floor_date(created_utc, "month"),
        year_month = format(created_utc, "%Y-%m"),
        # Combine pipeline-level sentiment (from transformer) with aspect-level sentiment (from LLM)
        # Dashboard can use either; keep both
        text_sentiment = sentiment_label,  # transformer-based, text-level
        aspect_sentiment = sentiment        # LLM-based, aspect-level (from ABSA unnesting)
      )

    # Summary
    summarise_pipeline(df_final)

    # Save
    saveRDS(df_final, file.path(CFG$path_processed, "08_full_pipeline_output.rds"))
    log_msg(paste("Phase 03 complete. Full pipeline output:", nrow(df_final), "rows |",
                  length(unique(df_final$id)), "unique records saved to 08_full_pipeline_output.rds"))
    ```

    Also add a `scripts/export_for_inference.R` shortcut:

